## IA (Information Architecture)

이 문서는 `docs/spec.md`의 5개 핵심 기능(대시보드/식단/운동/진행/설정)을 기준으로, 앱의 **정보구조(IA)**와 **라우팅/네비게이션 구조**를 확정합니다.

### 목표

- **모바일 퍼스트**: 하단 탭을 중심으로 1차 내비게이션을 구성합니다.
- **탭 = 최상위 정보구조**: 사용자가 “기록(식단/운동/지표)”을 빠르게 남기고, “요약(대시보드/진행)”을 확인하도록 설계합니다.
- **Public/Auth 분리**: 로그인 전/후 영역을 분리하고, `/app/*`는 보호합니다.

---

## 1) 사이트맵(전역)

- **Public (비로그인)**

  - `/` 랜딩
  - `/sign-in/*` 로그인
  - `/sign-up/*` 회원가입

- **App (로그인 후, 바텀탭)**

  - `/app/dashboard` 대시보드
  - `/app/meals` 식단 트래커
  - `/app/workouts` 운동 트래커
  - `/app/progress` 진행 상황(그래프)
  - `/app/settings/*` 프로필/설정

- **예외**
  - `*` NotFound

---

## 2) 라우팅 설계(권장)

### 라우트 테이블

| 영역   | 경로              | 화면            | 네비게이션 | 비고                           |
| ------ | ----------------- | --------------- | ---------- | ------------------------------ |
| Public | `/`               | Landing         | 헤더 링크  | 로그인 상태면 `/app` 진입 유도 |
| Public | `/sign-in/*`      | SignIn          | 헤더 링크  | Clerk 제공                     |
| Public | `/sign-up/*`      | SignUp          | 헤더 링크  | Clerk 제공                     |
| App    | `/app`            | (index)         | 탭         | `/app/dashboard`로 리다이렉트  |
| App    | `/app/dashboard`  | Dashboard       | 탭         | 오늘 요약 허브                 |
| App    | `/app/meals`      | Meal Tracker    | 탭         | 음식 기록/요약                 |
| App    | `/app/workouts`   | Workout Tracker | 탭         | 운동 기록/히스토리             |
| App    | `/app/progress`   | Progress        | 탭         | 그래프/커스텀 지표             |
| App    | `/app/settings/*` | Settings        | 탭         | 프로필/목표/앱설정             |

### 탭(바텀탭) 구성

- **Dashboard**: 홈
- **Meals**: 식단
- **Workouts**: 운동
- **Progress**: 변화
- **Settings**: 설정

> 탭은 “항상 접근 가능한 1차 내비게이션”이며, 탭 내부의 세부 기능은 화면 내 섹션/버튼으로 진입합니다.

## 3) 화면별 IA(섹션 구조)

각 화면은 “상단: 날짜/컨텍스트(오늘/선택한 날짜) + 주요 요약” → “중단: 기록/리스트” → “하단: 주요 액션(추가)” 흐름을 기본으로 합니다.

### 3.1 Dashboard (`/app/dashboard`)

- **상단**
  - 오늘 날짜 / 간단한 인사(선택)
- **일일 진행 상황 오버뷰**
  - 목표 대비 진행(예: 칼로리, 운동, 체중 입력 여부)
- **칼로리 섭취 현황(요약 카드)**
  - 오늘 총 섭취 kcal / 목표 kcal / 잔여
  - 매크로 요약(탄/단/지)
- **운동 요약(요약 카드)**
  - 오늘 운동 세션 유무
  - 총 볼륨/총 세트/총 시간(도입 단계에서는 최소 1~2개 지표부터)
- **주요 지표(사용자 선택)**
  - 체중, 체지방률, 주간 운동 횟수 등 2~4개 “핀”
- **빠른 액션**
  - 음식 추가 / 운동 기록 / 체중 입력

### 3.2 Meal Tracker (`/app/meals`)

- **상단**
  - 날짜 선택(기본=오늘)
  - 오늘 요약: 총 kcal + 매크로
- **식사 섹션**
  - 아침/점심/저녁/간식(또는 “식사 타입”)별 리스트
  - 각 항목: 음식명, 수량, kcal(매크로는 상세)
- **추가 플로우(핵심)**
  - 음식 검색(외부 DB or 로컬) → 항목 선택 → 수량/서빙 입력 → 저장
  - MVP 대안: 수동 입력(음식명 + kcal + 매크로)

### 3.3 Workout Tracker (`/app/workouts`)

- **상단**
  - 날짜 선택(기본=오늘)
  - 오늘 요약: 총 세트/총 볼륨(또는 운동 시간)
- **운동 기록(핵심)**
  - 운동 종목 리스트
  - 운동별 세트 테이블: set#, kg, reps
  - 빠른 입력 UX: “세트 추가” 버튼, 직전 세트 복사(추후)
- **운동 히스토리**
  - 날짜별 세션 목록(간단 리스트)

### 3.4 Progress (`/app/progress`)

- **상단**
  - 기간 선택(예: 1W/1M/3M/ALL)
  - 지표 선택(체중/체지방률/주요 리프트 등)
- **그래프 영역**
  - 선택 지표 라인 차트
- **기록 추가(핵심)**
  - 체중/체지방 등 값 입력
- **맞춤형 뷰**
  - “대시보드에 표시할 지표 핀” 선택(설정과 연결)

### 3.5 Settings (`/app/settings/*`)

- **카테고리 구조(권장)**
  - 프로필(Clerk)
  - 개인 정보(신체치수/나이 등)
  - 목표 설정(체중/칼로리/매크로/운동 목표)
  - 앱 설정(테마/알림 등)

> 현재는 Clerk `UserProfile` 중심이므로, 앱 고유 설정은 `settings` 내부의 별도 섹션/서브라우트로 확장합니다.

---

## 4) 핵심 유저 플로우(최소)

- **오늘 음식 기록**: Dashboard/Meals → 음식 추가 → 오늘 요약 반영
- **오늘 운동 기록**: Dashboard/Workouts → 운동 추가 → 세트 입력 → 오늘 요약 반영
- **체중/지표 기록**: Progress → 값 입력 → 그래프 반영 → 대시보드 주요 지표 반영
